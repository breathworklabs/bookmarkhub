# Unified Categorization System — Architecture Proposal

## Problem

6 disconnected grouping systems with overlapping responsibilities:

1. **Collections** — manual folders + smart collections (starred, recent, archived, uncategorized)
2. **Tags** — `bookmark.tags: string[]`, managed separately, auto-generated by SmartTaggingService
3. **Quick Filters** — 7 toggle buttons (starred, unread, broken, recent, archived, comments, engagement)
4. **Sidebar Navigation** — `activeSidebarItem` + `activeCollectionId` dual state
5. **Filter Presets** — saves all filter state as named presets
6. **Advanced Filters** — author, domain, content type, date range

Key issues:

- Starred/Recent/Archived exist in BOTH quick filters AND smart collections
- Tags are disconnected from collections — auto-tagging doesn't feed into groups
- SmartTaggingService has rich data but only produces tags, never auto-creates groups
- Sidebar navigation is overloaded — 'Collections' + activeCollectionId is a hack
- No unified concept of a "group"

## Solution: Single `View` Abstraction

### Core Types

```typescript
interface View {
  id: string
  name: string
  icon: string
  color: string
  parentId: string | null
  sortOrder: number

  mode: 'manual' | 'dynamic'

  // Manual: explicit bookmark membership
  bookmarkIds?: string[]

  // Dynamic: declarative criteria
  criteria?: ViewCriteria

  // UI
  pinned: boolean // visible in sidebar
  system: boolean // undeletable (All, Trash, etc.)

  createdAt: number
  updatedAt: number
}

interface ViewCriteria {
  // Boolean property filters (ex-quick-filters)
  starred?: boolean
  unread?: boolean
  broken?: boolean
  withComments?: boolean

  // Tags (ex-tag-filtering)
  tags?: string[]
  tagMatch?: 'any' | 'all'

  // Content filters (ex-advanced-filters)
  authors?: string[]
  domains?: string[]
  contentTypes?: string[]
  dateRange?: {
    preset?: 'today' | 'week' | 'month'
    start?: string
    end?: string
  }
  query?: string
  minEngagement?: number
  recentDays?: number

  // Exclusion
  excludeTags?: string[]
}
```

### Mapping: Old → New

| Current System                                | Becomes                                                                 | View Mode      |
| --------------------------------------------- | ----------------------------------------------------------------------- | -------------- |
| User Collections                              | Manual views with `bookmarkIds`                                         | `manual`       |
| Smart Collections (starred, recent, archived) | System dynamic views with criteria                                      | `dynamic`      |
| Quick Filters                                 | **Removed as system** — transient criteria overlay on active view       | N/A (UI state) |
| Filter Presets                                | Dynamic views with `pinned: false`                                      | `dynamic`      |
| Sidebar "All Bookmarks"                       | System view: `{ mode:'dynamic', criteria:{}, system:true }`             | `dynamic`      |
| Sidebar "Trash"                               | System view: `{ mode:'dynamic', criteria:{trashed:true}, system:true }` | `dynamic`      |
| Tags                                          | **Unchanged** — remain `bookmark.tags: string[]`                        | —              |
| SmartTaggingService                           | **Unchanged** — tags feed into dynamic view `criteria.tags`             | —              |

### Quick Filters → Transient Criteria Overlay

Quick filters become a UI-only overlay on top of the active view:

```typescript
// UI state only — not persisted as a view
interface ActiveFilterState {
  baseViewId: string // sidebar view we're inside
  overlay: Partial<ViewCriteria> // quick-filter toggles modify this
}
```

"I'm viewing my Dev collection AND I toggle starred" = manual view members filtered by overlay criteria.

### Store Structure

```typescript
interface UnifiedViewStore {
  views: Record<string, View>
  activeViewId: string
  membershipIndex: Map<string, Set<string>> // runtime: viewId → bookmarkId Set
  sidebarViewIds: string[] // ordered list of pinned view IDs

  // CRUD
  createView(partial: Partial<View>): View
  updateView(id: string, patch: Partial<View>): void
  deleteView(id: string): void

  // Membership (manual views only)
  moveToView(viewId: string, bookmarkIds: string[]): void
  removeFromView(viewId: string, bookmarkIds: string[]): void

  // Navigation
  setActiveView(id: string): void
  pinToSidebar(id: string): void
  unpinFromSidebar(id: string): void

  // Save current filters as a dynamic view
  saveAsView(name: string, criteria: ViewCriteria): View
}
```

### Filtering Pipeline

Single function, single pass, replacing all current filter logic:

```typescript
function getBookmarksForView(view: View, allBookmarks: Bookmark[]): Bookmark[] {
  if (view.mode === 'manual') {
    const memberSet = membershipIndex.get(view.id)
    return memberSet ? allBookmarks.filter((b) => memberSet.has(b.id)) : []
  }
  return allBookmarks.filter((b) => matchesCriteria(b, view.criteria!))
}
```

Performance unchanged: manual views use Set.has() (O(1)), dynamic views single-pass with early exits.

### What Gets Removed

- `collectionBookmarks` separate store → absorbed into View.bookmarkIds
- Smart Collections concept → system dynamic views
- Quick Filters as standalone system → transient criteria overlay
- `activeSidebarItem` / `activeCollectionId` dual state → single `activeViewId`
- Filter Presets as separate entity → unpinned dynamic views

### What Stays

- `TagManagerModal`, `TagInput` — unchanged
- `SmartTaggingService` (4 strategies) — unchanged, output feeds into dynamic view criteria
- Drag-drop tree UI — reused for view hierarchy
- Single-pass filter algorithm — unified `matchesCriteria` function
- localStorage persistence — new key `bookmarkhub_views_v2`

### Migration Strategy

- On load, check `_migrated_v2` flag in localStorage
- If absent, migrate: user collections → manual views, smart collections → system dynamic views, presets → unpinned dynamic views
- Set flag, keep old keys readable for one release cycle (rollback)
